# Evidence Pattern

## Purpose
Standardize the structure of evidence documents so audits are fast and comparable.

## When to Use
- Every batch closeout.
- Major architectural changes.

## When NOT to Use
- Trivial changes.

## Structure
- Executive Summary
- Files Changed
- Validation Results
- Blueprint Traceability
- Scope Lock Compliance
- Behavior Preservation
- Working Tree Inspection
- Final Audit Summary
- SA Final Review
- Closeout Decision

## Example
BATCH-1-CTA-CONTRACT-STANDARDIZATION.md and BATCH-2-COMMERCE-CONTEXT-STANDARDIZATION.md

## Anti-pattern
Ad-hoc evidence documents that omit key sections.

**Status**: Required for all batch closeouts.
