# ZENOVELL V4 Project Charter

## Purpose

Create a mobile-first landing page workspace for ZENOVELL V4 that is fast to iterate on, visually accountable, and simple enough for section-by-section delivery.

## Scope

- Next.js App Router frontend only
- TypeScript, Tailwind CSS, shadcn/ui
- One section = one deliverable
- Hero Section 1 is implemented and frozen as the current implementation baseline
- Next active implementation task is Section 2 only

## Out of Scope

- Backend, API, auth, payment, database
- Monorepo or package splitting
- Section 2 or later sections during Hero delivery

## Delivery Model

- Use the approved blueprint as implementation authority
- Keep content separate from presentation
- Require screenshot evidence for UI approval
- Treat lint, typecheck, and build as necessary but not sufficient
- Defer visual polish to the final site-wide polish phase unless explicitly directed otherwise

## Current State

- Workspace bootstrap complete
- Hero Section 1 implemented
- Hero frozen as implementation baseline
- Visual polish deferred to final site-wide polish phase
- Section 2 is the next active task
