## 2. Current LCP Analysis

LCP element on initial load: the full-bleed Hero background image inside `sections/hero/hero-section.tsx`.

- Container: `<section id="main-content" class="hero-root relative overflow-hidden bg-[#0A0A0A]">`
- LCP candidate: `<Image fill priority ... class="hero-background scale-[1.05] object-cover" ... />`
- Why LCP: Largest visible above-fold paint; text/CTA overlay depends on contrast from this image + scrims.
- MobileShell constrains to max-w-[430px]; sizes prop targets this.
- Preload generated by Next.js due to `priority`.
- No other images compete in Hero (TrustStrip icons are small decorative post-LCP).
- GlobalHeader above has no images.

Current risks before patch:
- Relied on implicit `priority` behavior for `fetchpriority="high"`.
- 365KB source JPEG (941×1672) — Next optimizes in prod.
- Container height driven by content flow (no explicit reserve for image box; mitigated by priority preload).
- 3 scrim layers add minor composite cost but required for legibility.

## 4. Safe Optimizations Applied

Only change:
- Added explicit `fetchPriority="high"` to the `<Image>` in `sections/hero/hero-section.tsx`.

Rationale (safe per scope):
- Explicit documents LCP intent.
- Reinforces the preload hint already produced by `priority`.
- Zero impact on layout, paint order, visual output, or DOM structure.
- No new props that could alter rendered pixels (no quality change, no placeholder, no objectFit change).

No other edits:
- Scrim gradients untouched (3 layers required for contrast).
- `scale-[1.05]` and inline styles untouched (visual parity).
- No CSS extraction or class changes.
- No changes to MobileShell, GlobalHeader, content/, layout, or any frozen section.
- No architecture changes (remains client component).

Before/After (source diff only):

```diff
 <Image
   src={BACKGROUND_IMAGE_SRC}
   alt=""
   aria-hidden="true"
   fill
   priority
+  fetchPriority="high"
   sizes="(max-width: 430px) 100vw, 430px"
   ...
 />
```

Build output confirmation (post-patch):
- `<link rel="preload" as="image" ... fetchPriority="high" imageSizes="..." />`
- `<img ... fetchPriority="high" decoding="async" ... />`
